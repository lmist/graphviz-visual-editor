## Design Context

### Users
The primary user is a human supervising agent-generated Graphviz output through MCP, with the option to step in and edit directly. They are working in modeling sessions where graphs are not static diagrams but living artifacts: generated, inspected, corrected, and refined as part of an active reasoning workflow.

### Brand Personality
Fast, sleek, chic. The interface should feel technically precise and polished without becoming ornamental. It should give the user confidence that agent activity is visible, editable, and under control.

### Aesthetic Direction
The product should feel like the sharper, cooler version of a modern diagramming workspace such as Mermaid's editor: a large, quiet canvas; immediate code access; lightweight panels; refined controls; and a sense of speed. Keep the established brutalist foundation: monochrome chrome, square edges, strong black borders, Space Grotesk for UI, JetBrains Mono for code, and one cyan accent. The graph surface can carry the visual complexity; the surrounding UI should stay crisp, calm, and efficient. Light mode is the current baseline, with dark mode expected eventually.

### Design Principles
- Make agent output inspectable first: graph changes, DOT source, sync state, and errors should be easy for a human to review at a glance.
- Preserve direct manipulation: every generated graph should remain editable through both the visual canvas and the DOT editor.
- Keep the workspace fast and dense: prioritize modeling flow over marketing-style presentation, explanatory copy, or decorative layout.
- Use polish through restraint: sharp alignment, disciplined spacing, precise typography, and purposeful controls should create the chic feel.
- Keep accessibility practical: preserve keyboard access, focus visibility, readable contrast, and motion settings that do not interfere with sustained modeling work.
